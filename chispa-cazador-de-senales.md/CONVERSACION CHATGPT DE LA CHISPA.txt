Thought for 14s

Sí. Basado en la documentación oficial actual de Upwork:

1. ¿Cuesta usar la API de Upwork?

No veo una tarifa publicada para usar la API en sí. Upwork dice que clientes y freelancers “en cualquier plan de membresía” pueden solicitar acceso a la API. Eso sugiere que no hay un costo directo por pedir/usar la API, pero sí requiere aprobación.

Lo importante:

Hay que solicitar una API key.
Upwork revisa la solicitud.
Responden por email, normalmente dentro de dos semanas.
La cuenta debe tener nombre válido, dirección, foto, método de pago activo/verificado e identidad verificada.
El límite diario permitido es 40,000 requests por día.
Upwork aclara que la API no está disponible para uso comercial. Esto es clave si quieren revender el sistema como SaaS o producto para terceros.

Mi lectura: para uso interno de tu equipo/agencia probablemente es viable, pero si la idea es vender una herramienta pública que monitoree jobs de Upwork para otros usuarios, ahí puede chocar con la restricción de “no commercial use”.

2. ¿Se puede tener un sistema automatizado/live que capture jobs con filtros?

Sí, pero con matices.

Upwork tiene queries GraphQL para buscar job posts del marketplace. La más relevante es marketplaceJobPostingsSearch, que permite buscar trabajos y aplicar filtros mediante marketPlaceJobFilter, además de usar opciones de ordenamiento con sortAttributes. Esa query requiere el permiso “Read marketplace Job Postings”.

También existe publicMarketplaceJobPostingsSearch, descrita como búsqueda de job postings públicos en el marketplace, usando filtros de búsqueda.

Entonces, para un sistema tipo “radar de oportunidades”, se podría hacer algo así:

Autenticarse con OAuth/API key.
Ejecutar búsquedas cada X minutos.
Filtrar por keywords, categoría, skills, presupuesto, país, experiencia, fecha, etc., según los campos disponibles.
Guardar los IDs ya vistos.
Detectar nuevos jobs.
Enviar alertas a Slack, email, Telegram, CRM, Notion, Airtable, Google Sheets, etc.

Sobre “live”: Upwork sí documenta Subscriptions in GraphQL, que funcionan con webhooks para eventos. La documentación menciona eventos de Job postings (JP) como NEW, UPDATE y CLOSE. Pero también dice que las subscriptions están disponibles solo para clientes y que cada subscription debe ser revisada y aprobada por el equipo de Upwork antes de quedar activa.

Mi conclusión práctica:

Opción A — Polling: más realista para empezar.
Consultar la API cada 5, 10 o 15 minutos con filtros. Esto no es 100% “live”, pero es suficientemente cercano para capturar nuevos jobs rápido, siempre respetando el límite de requests.

Opción B — Webhooks/subscriptions: más “live” de verdad.
Posible según la documentación, pero depende de que Upwork apruebe la subscription y de que tu cuenta/tipo de uso califique.

Resumen directo

Sí, se puede construir un sistema automatizado de captura de jobs con filtros usando la API de Upwork.
No parece tener un costo directo publicado, pero requiere aprobación, permisos correctos y respetar límites.
Para live real, usarías subscriptions/webhooks, pero eso necesita revisión/aprobación de Upwork.
Para producción rápida, yo empezaría con polling + base de datos + alertas.
